# Type Defender

> 양해 부탁드립니다: 이번 주는 제 실수로 구현한 기능을 한 번에 올리게 되었습니다. 다음 주부터는 기능별로 따로 브랜치 올리겠습니다.

## 구현 현황
### Notebook Input
- 노트 디자인으로 입력 창을 만들어 중앙 아래쪽으로 배치했습니다.
- 활성화된 단어 리스트에 있는 단어의 첫 글자를 입력할 시, 회색으로 가이드 텍스트가 표시됩니다.
- 정타는 초록, 오타는 빨강, 안 친 글자는 회색으로 표시됩니다.
- 정답을 5번 이상 맞출 시 콤보 판정으로 창의 끝쪽이 초록색으로 변합니다.
- 정답을 10번 이상 맞출 시 콤보 판정으로 창의 끝쪽이 노란색으로 변합니다.
- 정답 글자 타이핑 시, 글씨가 잠시 강조됩니다.
- 오답 글자 타이핑 시, 입력 창이 흔들립니다.

### HP Bar
- 오른쪽 상단, HP바가 위치합니다.
- 오답 시, HP가 20% 깎입니다.
- 65% ~ 100%는 ok(초록), 30 ~ 65%부터 warn(주황), 30% 아래로 떨어졌을 때 danger(빨강)으로 색이 변합니다.
- 0% 도달 시 게임 오버! 메시지를 띄웁니다.

### Toast
- 게임 플레이 중 도전과제 조건을 충족했을 때, 화면 상단 또는 측면에 팝업 형태로 알림을 띄웁니다.
-  gameClear()가 필수인 도전과제 5&6, 단어가 바닥에 닿았는지 판정이 필요한 도전과제 9를 제외한 나머지 도전과제 연동을 완료했습니다.
- 출현 후 약 2~3초 뒤에 자연스럽게 사라지도록 애니메이션을 적용했습니다.

### Grade Stamp
- 게임 오버 창에서 점수와 함께 표시할 스탬프입니다.
- 디자인 구현만 했고 실제로 사용하지 않았습니다. W2 게임 오버 창 개발 과정에서 연동할 예정입니다.

### Achievements
- 도전과제 12개를 정의했습니다.
- 도전과제 5&6, 도전과제 9를 제외한 나머지 도전과제 연동을 완료했습니다.

## 파일 구조
- index.html: 실제로 유저들에게 보여지는 플레이 화면
- css/style.css: 전체 디자인
- js/achievements.js: 도전과제 목록과 달성 여부, 출석 체크 등 플레이어의 업적 데이터를 관리
- js/game.js: 점수, HP, 레벨, 타자 속도(WPM) 등 게임의 핵심 진행 상태와 정답 및 오답 판정 로직을 제어
- js/input.js: 사용자의 실제 키보드 입력을 감지하고, 타이핑된 데이터를 UI와 게임 로직으로 전달
- js/main.js: 구성 요소들을 초기화하고 매 프레임마다 화면을 갱신하는 게임 루프를 실행
- js/ui.js: 화면에 입력된 글자의 색상 변화(정타/오타), 체력바 증감, 점수 갱신, 업적 달성 알림(토스트) 등 시각적인 화면 조작을 전담
- js/wordData.js: 게임 내 각 레벨별로 등장하는 타겟 영단어 데이터들이 분류되어 저장

## 실행 방법
index.html을 브라우저에서 열면 실제 화면을 확인할 수 있습니다. Live Server를 이용하여 실행하는 걸 권장드립니다.
